# 质量月活动策划技能

## 0. 能力说明与边界（调用前必读）

**本技能能做什么**
- 以**交互引导**方式，陪你从零策划一场质量月活动：先明确目标/主题/周期/范围/预算与重点活动类型，再据此展开成《质量月活动总体方案》。
- 将方案自动展开为**可执行的活动排期与清单**（阶段→日期→事项→责任方→产出）。
- 生成**宣传文案包**：主题口号、启动会致辞、海报文案、公众号推文（可联动你的内容管线）。
- 生成**总结与评优**材料：成效总结框架、关键指标对比、先进集体/个人评选标准与评分表。
- 输出**双版本**报告：结构化 Markdown 文档 + 精美网页版 HTML（含排期时间轴，适合对内宣贯与评审演示）。

**本技能不能做什么**
1. 不能替你拍板主题、目标、预算、重点活动——这些必须由你基于企业实际决定，Agent 只给维度、示例与选项，不替你下结论。
2. 不能替你编造活动数据或成效——总结报告中的指标（参与率、问题关闭率等）必须由你提供真实数据，Agent 不臆造。
3. 不能替代真实组织动员——本技能产出的是"方案与文案"，落地执行（开会、发奖、挂海报）仍需你与团队完成。
4. 不对方案做粉饰——若你给的预算极低或周期过短，Agent 会如实提示风险，不强行美化。
5. 不绑定特定质量管理体系标准——质量月是通用质量活动载体；若你做 ISO 9001/ IATF 16949 等专项，可叠加对应技能。

**领域说明**：质量月本身属质量管理活动，本技能**锁定质量领域**是合理的。但其"主题月活动策划"方法论可迁移到安全月、精益月等其他主题月——交互逻辑相通，你可按需替换主题词。

## 1. 调用前置：我需要的信息

开始前，请尽量提供以下信息（缺项可在交互中逐步补充，缺失项会标注「供参考·待确认」）：

| 信息 | 必填 | 说明 |
|------|------|------|
| 企业/组织名称 | 是 | 报告抬头与署名 |
| 质量月年份/届次 | 是 | 如"2026 年度""第 12 届" |
| 主题 | 是（可引导生成） | 如"夯实质量基础，筑牢发展根基" |
| 周期 | 是 | 起止日期，如"2026-09-01 至 2026-09-30" |
| 参与范围 | 是 | 如"全公司 12 个部门""仅制造体系" |
| 预算 | 否 | 总额或分项；缺失则方案标注「待确认」并提示影响 |
| 重点活动类型 | 否 | 如"QC 成果发表、质量知识竞赛、缺陷猎手"；缺失则 Agent 给活动库供选 |
| 往届参考 | 否 | 若有往届方案/总结，可作为对齐基线 |

**若信息不足**：Agent 会先以引导话术帮你逐项厘清（见第 2 节交互锁），不会直接代写。

### 1.1 输入载体判断（规则 2）
- 若你提供的是**纯文字描述**（如"我们想 9 月办质量月，主题还没定"）：直接进入交互引导，无需脚本解析。
- 若你提供的是**文档/表格**（如往届 Word 方案、Excel 排期表、PDF 总结）：Agent 会显式说明将调用文档处理技能（xlsx / docx / pdf / markitdown 等）抽取关键信息后再引导，不静默处理。
- 若最终需生成双版报告：调用本技能的 `scripts/build_report.py`，将结构化 JSON 渲染为 MD + HTML。

## 2. 交互锁流程（四门强制）

每门强制交互：Agent 给维度+质量领域示例 → 你判断/提供内容 → 若跳过则该部分标注「供参考·待确认」。

| 门 | 阶段 | Agent 引导你确认的核心内容 | 禁止 |
|----|------|---------------------------|------|
| 门1 | 方案策划 | 目标（可多目标）、主题、周期、范围、预算、重点活动类型、组织机制 | 替你定主题/预算 |
| 门2 | 活动排期 | 将重点活动拆为阶段（启动/开展/高潮/收尾），每阶段事项、日期、责任方、产出 | 替你编日期/责任方 |
| 门3 | 宣传文案 | 口号调性、致辞要点、海报诉求、推文角度（可联动内容管线） | 替你定对外口径 |
| 门4 | 总结评优 | 成效指标口径、评优奖项设置、评选标准与评分维度 | 替你编指标数据 |

**缺失信息标注纪律（规则 3）**：任一门你暂无法提供的内容，Agent 主动说明"缺这项会导致方案哪部分失真"，并在该部分标注 **「供参考·待确认」**；你执意不提供时绝不伪造成已确认内容。例如未给预算，则预算章节标「待确认」并提示"无法评估活动规模可行性"。

## 3. 输出形式（规则 4：先大纲后文档）

四门收齐（或你确认以「供参考」占位）后，**先展示报告大纲**确认，再生成双版本：

- `质量月活动策划_报告.md`
- `质量月活动策划_报告.html`（精美网页版，含排期时间轴）

**报告大纲（章节结构）**
1. 封面与元信息（企业/年份/主题/周期/范围/预算）
2. 一、活动总体方案（背景、目标、原则、组织机制）
3. 二、活动排期与清单（阶段时间轴 + 活动明细表）
4. 三、宣传文案包（主题口号、启动会致辞、海报文案、公众号推文）
5. 四、总结与评优（成效框架、关键指标对比、评优标准与评分表）
6. 五、供参考·待确认项（汇总所有占位内容）

> 大纲确认后 Agent 才生成正文；你也可要求先只出某一章（如只先出方案）。

## 5. 相关技能联动（质量活动线）

质量月是"活动运营类"入口，与现有质量工具/分析技能构成互补：

- **总结阶段数据沉淀** → `质量月报/工作周报` 技能（将质量月成效指标转为周期性月报）
- **QC 小组 / QCC 课题** → `QC七大手法`、`8D` 技能（活动落地的方法论支撑）
- **单项活动落地（竞赛 / 培训）** → `quality-knowledge-quiz`（质量知识竞赛技能）、`quality-training-planner`（质量培训策划技能）：质量月方案中的"知识竞赛""培训"活动条目，分别由这两个技能承接出可落地包
- **评优中的质量问题归因** → `鱼骨图5M1E`、`质量改进优先级排序`（后续开发候选）可作为下游 feeder
- **与战略工具链关系**：质量月是"执行层活动"，战略工具链（PESTEL→五力→VRIO→SWOT→SMART→质量目标分解）是"规划层"；质量月的目标可与 `qms-objective-decomposition` 的质量目标对齐，但本技能不重复做目标分解。
- **关系说明**：本技能专注"办一场质量月"的全周期策划与文案，不做质量统计分析、不做体系认证，相关深水区交给对应技能。
